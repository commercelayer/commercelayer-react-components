# Prices

The Prices components let you fetch and display product prices from the Commerce Layer API.
All price components must be nested inside the `<CommerceLayer>` context that handles API authentication.

Refer to the [Prices API reference](https://docs.commercelayer.io/core/v/api-reference/prices/object)
for the full list of available attributes.

---

## PricesContainer

`PricesContainer` is the main container for price data. It fetches prices for one or more SKU codes
and stores them in a React context for its children.

Multiple `<Price>` children each register their own `skuCode` тАФ the container automatically
**batches all registrations into a single API request** using a 50ms debounce.

<span title='Requirements' type='warning'>
Must be a child of the `<CommerceLayer>` component.
</span>

<span title='Children' type='info'>
`<Price>`
</span>

**Props**

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `skuCode` | `string` | тАФ | SKU code to fetch price for (used when there is a single `<Price>`) |
| `filters` | `object` | тАФ | Optional Commerce Layer price filters (e.g. `{ currency_code_eq: 'EUR' }`) |
| `perPage` | `number` | тАФ | Max prices per page (default: `10`) |

<Source
  language="jsx"
  dark
  code={`
import { CommerceLayer, PricesContainer, Price } from '@commercelayer/react-components'

<CommerceLayer accessToken="..." endpoint="https://yourdomain.commercelayer.io">
  <PricesContainer skuCode="MY-SKU-CODE">
    <Price />
  </PricesContainer>
</CommerceLayer>
`}
/>

## Batched prices (single API request)

Mount multiple `<Price>` components тАФ each registers its own `skuCode` and the container
debounces all registrations into **one API call**.

<span title='Performance tip' type='info'>
No matter how many `<Price>` components you add, only a single request is sent to the API.
</span>

<Source
  language="jsx"
  dark
  code={`
<PricesContainer>
  <Price skuCode="SKU-A" />
  <Price skuCode="SKU-B" />
  <Price skuCode="SKU-C" />
</PricesContainer>
`}
/>

## Price

`Price` renders the `formatted_amount` and optionally the `formatted_compare_at_amount` for a SKU.
It reads prices from the parent `PricesContainer` context.

<span title='Requirements' type='warning'>
Must be a descendant of `<PricesContainer>`.
</span>

**Props**

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `skuCode` | `string` | тАФ | SKU code (overrides the container's `skuCode`) |
| `showCompare` | `boolean` | `true` | Whether to show `formatted_compare_at_amount` |
| `compareClassName` | `string` | тАФ | CSS class applied to the compare-at price element |
| `children` | `function` | тАФ | Render prop: `({ prices, loading }) => ReactNode` |

<Source
  language="jsx"
  dark
  code={`
<PricesContainer skuCode="MY-SKU-CODE">
  <Price
    className="font-bold text-xl"
    compareClassName="line-through ml-2 text-gray-400"
  />
</PricesContainer>
`}
/>

## Render prop

Use the `children` render prop to access the raw `prices` array and `loading` state
for a fully custom price UI.

<Source
  language="jsx"
  dark
  code={`
<PricesContainer skuCode="MY-SKU-CODE">
  <Price>
    {({ prices, loading }) => {
      if (loading) return <span>LoadingтАж</span>
      if (prices.length === 0) return <span>No price available</span>
      const [p] = prices
      return (
        <div>
          <strong>{p.formatted_amount}</strong>
          {p.formatted_compare_at_amount != null && (
            <s className="ml-2 text-gray-400">{p.formatted_compare_at_amount}</s>
          )}
        </div>
      )
    }}
  </Price>
</PricesContainer>
`}
/>
